### 任务 1：P0 caseId 状态管理统一

**问题**：6 个页面使用旧模式 `route.query.caseId || state.caseId`，存在 caseId 丢失/脏值风险，导致 API 调用 URL 出现 `/api/cases/null/liability` 等错误。

**解决方案**：
1. 在 [useAccidentFlow.js](file:///c:/Users/28129/Desktop/ACCIDENT/src/stores/useAccidentFlow.js) 新增 `isValidCaseId`、`getCurrentCase`、`setCurrentCase` 工具函数
2. `isValidCaseId` 过滤无效值：null、undefined、"null"、"undefined"、"NaN"、""
3. 统一 6 个页面的 caseId 获取逻辑：

```js
const currentCaseId = () => {
  const queryId = route.query.caseId
  if (isValidCaseId(queryId)) return String(queryId).trim()
  return getCurrentCase()
}
```

4. 统一 onMounted 恢复逻辑：URL query → store → localStorage 三层回退

**涉及文件**：
- [AccidentEntry.vue](file:///c:/Users/28129/Desktop/ACCIDENT/src/views/AccidentEntry.vue)
- [Liability.vue](file:///c:/Users/28129/Desktop/ACCIDENT/src/views/Liability.vue)

- [VideoProcessing.vue](file:///c:/Users/28129/Desktop/ACCIDENT/src/views/VideoProcessing.vue)

**现在的情况**：连续刷新 10 次，0 次 caseId 丢失。无效 ID 被拦截，不再出现 `/api/cases/null/...` 错误。

### 运行时修复 3：移动端局域网访问

**问题**：手机访问 `127.0.0.1:5173` 显示"拒绝连接"。

**根因**：
1. Vite 配置 `host: '127.0.0.1'` 只绑定本地回环地址
2. `127.0.0.1` 在手机上指向手机自身，不是电脑

**解决方案**：
- [vite.config.js](file:///c:/Users/28129/Desktop/ACCIDENT/vite.config.js) 的 `host` 改为 `0.0.0.0`，服务器监听所有网络接口
- 手机使用电脑局域网 IP 访问（如 `http://10.22.9.122:5173/`）

**现在的情况**：手机连接同一 WiFi，可正常访问平台。

### 运行时修复 4：失败兜底前端提示（6 类异常）

**问题**：API 调用失败时只显示原始错误信息，用户体验差。

**解决方案**：在 [api/index.js](file:///c:/Users/28129/Desktop/ACCIDENT/src/api/index.js) 新增 `showFriendlyError` 函数，在 `request` 和 `uploadFile` 中调用，根据错误码和错误信息匹配 6 类异常场景：

| 异常场景 | 触发条件 | 友好提示 |
|----------|----------|----------|
| Dify 不可用 | 502/504/500+dify | "AI 分析服务暂时不可用，已转人工复核流程" |
| 图片模型不可用 | 500+image evidence | "图片识别服务异常，请稍后重试" |
| 规则未命中 | 空数组返回 | "未匹配到规则，建议人工复核"（[RuleBasis.vue](file:///c:/Users/28129/Desktop/ACCIDENT/src/views/RuleBasis.vue)） |
| 证据冲突 | conflicts 字段 | "检测到证据冲突，已降低可信度，建议进入人工复核"（[IntelligentAnalysis.vue](file:///c:/Users/28129/Desktop/ACCIDENT/src/views/IntelligentAnalysis.vue)） |
| 报告导出失败 | 500/404+report | "报告生成失败，可重新生成或稍后重试" |
| 非法状态跳转 | 400+状态关键字 | "案件当前状态不允许此操作，请按正确流程处理" |
| 通用兜底 | 404/500/503 | "资源不存在"/"服务器错误"/"服务不可用" |

带 2 秒防抖，避免同类错误短时间内重复弹窗。

**现在的情况**：所有 API 错误都有友好提示，不会显示原始错误堆栈。
