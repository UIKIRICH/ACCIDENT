# 视频语义校验层 · 前端修改方案

> 基于 `新任务.md` 中"检测—语义双通道视频证据理解机制"的整体架构改造，针对**前端 + 接口对接**部分制定的具体执行方案。
> 后端三个核心接口（`video-semantic-check` / `fuse-video-evidence` / `fused-evidence`）及融合规则 `fuse_video_evidence` 已实现，本方案聚焦前端对接与展示改造。

---

## 一、整体架构变化

### 原架构
```
视频上传 → 神经网络视频检测模型 → Dify 责任推理 → 报告
```

### 新架构
```
视频上传 → 关键帧抽取 → 原视频检测模型 → 千问视频语义校验模型
       → 证据融合与冲突门控 → 结构化证据包 → Dify 责任推理 → 报告
```

**职责边界**：
- 检测模型：负责"看到了什么"（车辆目标、关键帧、事故时间点）
- 千问语义校验：负责"这段视频在发生什么"（视角识别、自车隐式参与、车辆行为语义）
- 融合层：负责"这些证据能不能进入责任推理"（冲突门控、人工复核判定）
- Dify：基于融合证据包生成责任说明，不直接定责

## 四、前端修改方案（按优先级）

### P0 · API 层接口对接（基础，必须先做）

**文件**：`src/api/index.js`

**改动**：在 `CasesAPI` 对象中新增 3 个方法，对应后端已实现的 3 个接口。

```javascript
// 在 CasesAPI 对象中新增：

// 1. 获取融合证据包（前端展示用，最常用）
async getFusedEvidence(caseId) {
  return request('GET', `/api/cases/${caseId}/fused-evidence`)
},

// 2. 触发千问视频语义校验（可选，通常由视频处理流程自动触发）
async videoSemanticCheck(caseId, data) {
  // data: { video_path, keyframes, detector_output }
  return request('POST', `/api/cases/${caseId}/video-semantic-check`, data)
},

// 3. 触发证据融合（可选，通常由后端流程自动调用）
async fuseVideoEvidence(caseId, data) {
  // data: { detector_output, qwen_semantic_check }
  return request('POST', `/api/cases/${caseId}/fuse-video-evidence`, data)
}
```

**说明**：前端**不直接调用千问 API**，只调自己后端，避免 API Key 暴露。前端主要使用 `getFusedEvidence` 拉取已融合的证据包进行展示；`videoSemanticCheck` 和 `fuseVideoEvidence` 一般由视频处理流程在后端自动串联完成，前端仅在需要手动重试时调用。

检测模型候选类型：追尾事故，置信度 0.333
语义校验结果：疑似前车变道 / 切入 / 侧向接触
最终事故类型：暂不直接认定

```
追尾特征支持：未通过

```

```
证据一致性：偏低，0.469
解释：系统将关键帧检测结果、视频检测模型输出和千问语义校验结果进行比对。
当前检测模型给出追尾候选，但追尾特征支持不足，且语义校验更倾向于前车变道或切入关系，
因此系统判定证据存在不一致，进入人工复核。
```

## 九、风险与注意事项

1. **后端接口可能返回空 `fused_evidence_packet`**：当案件未跑过语义校验流程时，`GET /fused-evidence` 返回 `{}`。前端必须做空值兜底，所有字段访问使用可选链 `?.` 并提供默认值。

2. **字段枚举值可能扩展**：千问返回的 `camera_view`、`external_vehicle_behavior` 等枚举值未来可能新增，前端中文映射表应提供 `unknown` 兜底，避免新值导致展示异常。

3. **不要在前端硬编码千问 API Key**：前端只调后端接口，不直接调千问。`videoSemanticCheck` 方法虽提供，但仅在需要手动重试时使用，常规流程由后端自动串联。

4. **保留旧字段兼容**：`vehicle_count`、`accident_type` 等旧字段在后端可能仍存在，前端展示时优先使用融合证据包的新字段，旧字段仅作 fallback。

5. **Dify 结果展示不变**：`IntelligentAnalysis.vue` 中的 Dify 分析结果展示区块（`dify-analysis-container`）保持不变，Dify 已在后端改为读取融合证据包，前端无感知。
