## 二、生产部署支持

### 2.1 FastAPI 直接托管前端静态文件（方案 B）

`main.py` 末尾新增：若 `dist/` 目录存在且 `SERVE_STATIC` 环境变量为 `true`（默认），自动将前端静态文件挂载到根路径。

- 环境变量 `SERVE_STATIC=true` → 启用（默认）
- 环境变量 `SERVE_STATIC=false` → 跳过（留给 Nginx 代理）

### 2.2 CORS 来源可配置

`main.py` 的 CORS 中间件从硬编码 `allow_origins=["*"]` 改为读取 `CORS_ALLOW_ORIGINS` 环境变量：
- 已设置：按逗号分隔的域名列表限制来源
- 未设置（开发模式）：默认 `["*"]`

### 2.3 Nginx 反向代理配置参考

新建 `nginx.conf.example`，包含完整的 Nginx 配置模板（方案 A），涵盖：
- SPA 路由 fallback
- `/api/`、`/health`、`/keyframes`、`/uploaded_videos`、`/uploaded_images`、`/temp` 代理
- 上传大小限制
- HTTPS 提示

### 2.4 `outputs/` 目录在启动时创建

`main.py` 启动阶段新增 `OUTPUTS_DIR.mkdir(parents=True, exist_ok=True)`，与 `temp/`、`keyframes/` 等目录行为保持一致。

## 五、新增功能

### 5.1 规则详情接口

`main.py` 新增 `GET /api/rules/{rule_id}`，原后端缺少此接口（仅有列表、创建、修改、删除），前端无法查询单条规则详情。

- 权限：登录用户即可访问
- 不存在的规则返回 404

### 5.2 YOLO 图片证据分析

`backend/video_keyframe.py` 新增 `analyze_image_with_yolo()` 函数，使用已有 YOLO 模型分析单张事故图片：

- 检测图片中车辆位置和数量
- 分析前后车水平重叠度判断追尾关系
- 输出结构化证据包（追尾似然度、置信度、一致性标签等）

`main.py` 中两个图片证据接口改用此函数，**不再依赖 `image_evidence_demo/` 外部目录和 `accident_classifier` 模块**：

| 接口 | 说明 |
|------|------|
| `POST /api/analyze_image_evidence/` | 通过 URL 指定已存在的关键帧图片 |
| `POST /api/analyze_image_file_evidence/` | 上传新图片文件，保存到 `uploaded_images/` 目录 |

## 七、已确认无需修改的项目

| 项目 | 原因 |
|------|------|
| `BASE_DIR = Path(__file__).parent.absolute()` | 动态获取，不依赖硬编码路径 |
| SQLite 数据库路径 | 通过 `DATABASE_URL` 环境变量配置，默认相对路径 |
| `.env` 加载路径 | `load_dotenv(BASE_DIR / ".env")` 正确 |
| `SERVE_STATIC` 静态文件挂载位置 | 在所有 API 路由之后、`uvicorn.run` 之前，路由优先级正确 |
| Vue Router base URL | 空值适用于根路径部署 |
| `vite.config.js` proxy | 仅开发环境使用，生产环境不生效 |
